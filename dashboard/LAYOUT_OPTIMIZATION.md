# 页面布局优化方案

## 当前问题分析

1. **网格布局不够灵活**：卡片大小不一致，空间利用不充分
2. **健康度面板占用空间大**：横向布局可以更紧凑
3. **卡片分组不清晰**：相关功能分散，缺乏逻辑分组
4. **头部工具栏拥挤**：搜索框、按钮等元素排列可以优化
5. **卡片内容区域空白多**：部分卡片内容少但占用空间大
6. **缺少功能分组**：可以考虑使用标签页或侧边栏

## 优化方案

### 1. 改进网格布局系统
- 使用更智能的网格模板，支持不同大小的卡片
- 关键信息卡片（健康度、系统概览）占据更大空间
- 列表类卡片（Agent、任务）使用统一大小

### 2. 优化健康度面板
- 改为更紧凑的横向布局
- 健康度分数、状态、问题列表横向排列
- 减少垂直空间占用

### 3. 卡片功能分组
- **系统监控组**：健康度、系统概览、资源监控
- **Agent管理组**：Agent列表、模型配额
- **任务管理组**：当前任务、任务历史
- **通道监控组**：通道状态
- **工具组**：快捷操作、日志

### 4. 优化头部布局
- 左侧：标题和副标题
- 中间：全局搜索（更突出）
- 右侧：工具栏按钮（主题、全屏、通知、刷新）紧凑排列

### 5. 改进卡片内容布局
- 减少不必要的padding
- 内容区域使用更紧凑的间距
- 图表和列表更好地利用空间

### 6. 响应式优化
- 大屏（>1400px）：3-4列网格
- 中屏（768-1400px）：2列网格
- 小屏（<768px）：1列网格，卡片堆叠

## 实施步骤

1. 优化CSS网格布局
2. 重构健康度面板布局
3. 优化头部工具栏
4. 改进卡片内容间距
5. 增强响应式布局
